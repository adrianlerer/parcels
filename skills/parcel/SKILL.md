# Parcel — hand off work to another machine

The `parcel` CLI lives at `~/ai/parcels/bin/parcel` (also on PATH as `parcel`).
Targets: run `parcel targets` to list (e.g. `pop-os`, `spark`).

## Steps

1. **Write intent into `HANDOFF.md`** at the repo root before shipping. This is
   the most valuable part of the handoff — the remote agent (possibly a different
   one) reads it first. Include: what you were working on, current state, concrete
   next steps, known gotchas, and how to verify. Keep any existing
   "## Machine state (auto-generated by parcel" section — parcel regenerates it.

2. **Ship it.** From inside the repo:
   ```
   parcel send <target> --agent claude --session <your-session-id>
   ```
   - Your session id is the UUID in your scratchpad path (`.../<uuid>/scratchpad`).
     If unsure, omit `--session` — parcel picks the newest session for this cwd.
   - `--agent codex|pi|droid` for other agents' sessions.
   - `--prompt "..."` to steer what the remote agent does first (default:
     continue per HANDOFF.md).
   - `--idle` to ship without launching.

3. **Verify it's actually running** (never assume): `parcel status <target>`
   should show the tmux pane with the remote agent producing output. If launch
   failed, run `parcel doctor <target>` and report what's missing.

4. Tell the user how to attach: `ssh -t <host> tmux attach -t parcel-<repo>`.

## If doctor reports missing auth
- claude: user runs `claude login` on the target once, or `claude setup-token` here + set `CLAUDE_CODE_OAUTH_TOKEN` there.
- codex/pi: `parcel auth <target>` copies the credential files.
- droid: user runs `droid` once on the target.
